## <img src="https://api.iconify.design/lucide:calendar-check.svg?color=%2334d399" width="22" height="22" alt="" /> Visao Geral

**AgendaPro** e uma plataforma SaaS de agendamentos online para negocios que precisam vender, configurar e operar uma agenda profissional sem depender de planilhas, mensagens soltas ou configuracoes tecnicas expostas ao cliente final.

O projeto combina uma experiencia publica guiada para agendamento, um painel privado para o cliente operar sua propria agenda e uma **Central Dev** para suporte, auditoria, saude operacional, pagamentos, implantacao, logs, keys e diagnostico de producao.

O visual segue uma identidade **dark premium**, emerald/neon, executiva e agora com um tom **rustico sofisticado**: contraste forte, metal escuro, verde vivo, detalhes terrosos e movimento leve.

## <img src="https://api.iconify.design/lucide:layers-3.svg?color=%2334d399" width="22" height="22" alt="" /> Stack

### <img src="https://api.iconify.design/lucide:monitor-smartphone.svg?color=%2361dafb" width="18" height="18" alt="" /> Frontend

- **React** com componentes funcionais e estado local.
- **Vite** para desenvolvimento, build e preview.
- **TypeScript** com `strict` ativo.
- **Framer Motion** para microinteracoes e transicoes leves.
- **Lucide React** para iconografia consistente.
- **Recharts** para visualizacoes e relatorios.
- **QRCode React** para utilitarios de link/agenda.
- **CSS customizado** com visual dark premium, emerald/neon, responsividade e `prefers-reduced-motion`.

### <img src="https://api.iconify.design/lucide:server-cog.svg?color=%23fbbf24" width="18" height="18" alt="" /> Backend

- **Node.js 20.x**.
- **CommonJS** preservado com `require` e `module.exports`.
- **Vercel Serverless Functions** em `api/`.
- Endpoints internos em `server/endpoints/`.
- Sessoes assinadas para cliente e dev.
- Headers de seguranca centralizados.
- Tratamento de erro amigavel para usuario e mais tecnico apenas onde faz sentido.

### <img src="https://api.iconify.design/lucide:database.svg?color=%233ecf8e" width="18" height="18" alt="" /> Dados e Integracoes

- **Supabase REST** como camada de dados operacional.
- **Supabase Service Role** somente no backend.
- **Mercado Pago** para checkout e webhooks.
- **Vercel Environment Variables** como fonte oficial de configuracao.
- **Playwright** para testes E2E quando o navegador local estiver instalado.

## <img src="https://api.iconify.design/lucide:terminal.svg?color=%23d6a15f" width="22" height="22" alt="" /> Comandos

Instalar dependencias com registry publico:

```powershell
npm install --registry https://registry.npmjs.org/ --no-audit --no-fund
```

Rodar em desenvolvimento:

```powershell
npm run dev
```

Gerar build de producao:

```powershell
npm run build
```

Visualizar build local:

```powershell
npm run preview
```

Rodar E2E:

```powershell
npx playwright install
npm run test:e2e
```

## <img src="https://api.iconify.design/lucide:wrench.svg?color=%23fbbf24" width="22" height="22" alt="" /> Utilitarios Operacionais

- `npm run build` valida TypeScript e build Vite.
- `node --check api/dev.js` valida sintaxe CommonJS em endpoint serverless.
- `node --check server/endpoints/dev-dashboard.js` valida a Central Dev.
- `npm audit --registry https://registry.npmjs.org/ --omit=dev` checa vulnerabilidades de producao.
- `git diff --check` evita whitespace problematico antes do commit.
- `npx playwright test tests/publico.spec.ts --project=chromium` valida fluxo publico quando o Chromium local estiver instalado.

## <img src="https://api.iconify.design/lucide:lock-keyhole.svg?color=%23fb7185" width="22" height="22" alt="" /> Seguranca e Producao

- Nao versionar `.env` real, secrets, tokens, service role, uploads privados ou banco real.
- Nao expor `SUPABASE_SERVICE_ROLE_KEY`, `CLIENT_SESSION_SECRET`, `DEV_ADMIN_SECRET`, `DEV_ADMIN_PASSWORD` ou tokens Mercado Pago no frontend.
- Manter configuracao sensivel em **Vercel Environment Variables**.
- Manter backend em **CommonJS**.
- Manter Node **20.x**.
- Evitar SQL novo sem necessidade operacional clara.
- Proteger permissoes no backend e nas rotas, nao apenas escondendo botoes.
- `README.md` e `TESTES_E2E.md` nao devem entrar em ZIP de entrega quando a regra da sprint pedir pacote limpo.
